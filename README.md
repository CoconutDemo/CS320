# How can I ensure that my code, program, or software is functional and secure?
Some ways that I could ensure that my code, program, or software is functional and secure is by implementing automated testing. These tests could include testing individual or components interacting with each other. I would also enforce thorough examinations and collaborations with peer code reviews. Having all developers on the same page of practicing secured coding practices ensures that we are keeping security and high quality of coding early on.

# How do I interpret user needs and incorporate them into a program?
One way I would interpret user needs would be by having one-on-one conversations or interviews with users about their goals and interests in the software program. This would include having a general idea of the client's intent behind the project and also asking questions and clarifying subjects in the interview. I can then document the entire conversation and use it as a base in the development project and work on creating goals by meeting the user needs.

# How do I approach designing software?
After creating the requirements document, we can now move into the design phase of the software creation. We can now design and organize different components and functions within the software and start by smaller units and work from there. We can construct the data structure and work on test cases on expected outputs to ensure proper functionality. Feedback is most important, so actively seeking others input or criticism would be most beneficial to ensure top quality within the program.
